# External Integrations

**Analysis Date:** 2026-04-29

## APIs & External Services

**Scientific compute tools:**
- GROMACS - receptor preparation, solvation, equilibration, production MD, and trajectory conversion
  - SDK/Client: CLI binaries invoked from `scripts/infra/common.sh`, `scripts/rec/0_prep.sh`, `scripts/com/0_prep.sh`, `scripts/com/1_pr_prod.sh`, and `scripts/com/2_trj4mmpbsa.sh`
  - Auth: Not applicable
- gnina - docking engine for `blind`, `targeted`, and `test` modes
  - SDK/Client: CLI binary invoked by `scripts/dock/2_gnina.sh`
  - Auth: Not applicable
- gmx_MMPBSA - MM/PBSA binding-energy engine
  - SDK/Client: CLI binary invoked by `scripts/com/2_mmpbsa.sh`
  - Auth: Not applicable

**Scheduler / execution infrastructure:**
- Slurm - asynchronous job submission and status checks for heavy stages
  - SDK/Client: `sbatch`, `squeue`, and `sacct` used in `scripts/infra/common.sh`, `scripts/rec/0_prep.sh`, `scripts/dock/2_gnina.sh`, `scripts/com/1_pr_prod.sh`, and `scripts/com/2_sub_mmpbsa.sh`
  - Auth: Cluster login/session outside the repository; no token handling is implemented in `scripts/`

**Chemistry / structure utilities:**
- PDB2PQR - receptor protonation before GROMACS topology generation
  - SDK/Client: `pdb2pqr` or `pdb2pqr30` selected in `scripts/rec/0_prep.sh`
  - Auth: Not applicable
- Open Babel - optional SDF→GRO conversion path
  - SDK/Client: `obabel` invoked by `scripts/dock/0_sdf2gro.sh`
  - Auth: Not applicable

**Agent/plugin platform:**
- OpenCode plugin runtime - agent-facing plugin execution for workspace init, preflight, handoff inspection, group-ID checks, and conversion caching
  - SDK/Client: `@opencode-ai/plugin` used in `.opencode/plugins/*.js`
  - Auth: No provider credentials detected in `.opencode/package.json`, `.opencode/plugins/package.json`, or `opencode.json`

## Environment Configuration

**Required env vars:**
- `CONDA_DEFAULT_ENV` - operationally expected to resolve to `autoEnsmblDockMD` after sourcing `scripts/setenv.sh`
- `SLURM_JOB_ID` - used for backend detection in `scripts/infra/config.py` and `scripts/infra/executor.py`
- `SLURM_TMPDIR` and `SLURM_NODELIST` - also used by `scripts/infra/executor.py` to detect Slurm execution context
- `SECTION_KEY`-style overrides generated by `scripts/infra/config_loader.sh`; use variables such as `GENERAL_WORKDIR`, `SLURM_PARTITION`, `DOCKING_MODE`, and `MMPBSA_MPI_RANKS` when overriding INI values from the environment
