<task type="auto">
  <name>Task 1: Create bash config loader</name>
  <files>scripts/infra/config_loader.sh</files>
  <action>
Create a bash library script that reads INI-format config files (same format used by Python ConfigManager in scripts/infra/config.py). Must provide:

1. `load_config <file>` - Parse INI file, store values in associative array
2. `get_config <section> <key> [default]` - Retrieve config value with optional default
3. `require_config <section> <key>` - Get config value or exit with error
4. `config_has <section> <key>` - Check if key exists (return 0/1)

Support:
- Comments (lines starting with # or ;)
- Section headers [section_name]
- key = value pairs (trim whitespace)
- Environment variable override: `SECTION_KEY` overrides config file value (uppercase section + underscore + uppercase key)

Pattern: Pure bash (no external deps), sourceable (no main execution block), use `declare -gA` for associative arrays.

Test with: `source scripts/infra/config_loader.sh && load_config test.ini && get_config slurm partition default_val`
  </action>
  <verify>Create a minimal test.ini with [slurm] partition=rtx4090, source the loader, verify get_config returns correct values</verify>
  <done>config_loader.sh sourceable by any bash script, reads INI format, supports defaults and env overrides</done>
</task>

<task type="auto">
  <name>Task 2: Create common shell utilities</name>
  <files>scripts/infra/common.sh</files>
  <action>
Create a bash library with shared utilities for all workflow scripts. Must provide:

1. **Logging:** `log_info`, `log_warn`, `log_error` - prefixed with timestamp and level, stderr for errors
2. **Validation:** `require_file <path> [msg]`, `require_dir <path> [msg]`, `require_cmd <cmd>` - exit 1 if missing
3. **GROMACS helpers:** `run_gmx <args...>` - wraps gmx command with error checking and logging
4. **Slurm helpers:** `submit_job <script>` - submits and captures job ID, `wait_job <job_id>` - polls until completion
5. **Directory helpers:** `ensure_dir <path>`, `backup_if_exists <file>`
6. **Script header:** `init_script <name> [config_file]` - sources setenv.sh, loads config if provided, logs script start

Pattern: Pure bash, sourceable. When sourced, auto-sources config_loader.sh from same directory.

Reference: Look at patterns in expected/chm/BRD4/scripts/ and expected/amb/scripts/ for the common operations that should be abstracted.
  </action>
  <verify>Source common.sh, verify log_info prints timestamped message, require_file exits on missing file, run_gmx wraps gmx command</verify>
  <done>common.sh provides reusable utilities, auto-sources config_loader.sh, all functions work standalone</done>
</task>
