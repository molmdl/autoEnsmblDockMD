**Class: AnalyzerAgent(BaseAgent)**

- `execute(self, input_data: Dict) -> Dict`:

  2. Determine which analysis scripts to run based on stage (use STAGE_ANALYSIS_MAP)
  3. Run each analysis script via _run_analysis()
  4. Optionally run custom analysis hooks via _run_custom_hooks()
  5. Aggregate results into HandoffRecord and return

- `STAGE_ANALYSIS_MAP`: Class-level dict mapping stage names to lists of analysis script paths. For example:
  - "complex_analysis": ["scripts/com/3_ana.sh"]

- `_run_analysis(self, script: str, params: Dict) -> Dict[str, Any]`:

- `_run_custom_hooks(self, stage: str, data: Dict) -> List[Dict[str, Any]]`:
  - Look for custom analysis scripts in workspace / 'custom_analysis' / stage /
  - Execute each .py or .sh script found
  - Return list of result dicts
  - If no custom hooks directory or no scripts, return empty list

- `_collect_output_files(self, output_dir: Path, patterns: List[str] = None) -> List[str]`:
  - Glob for common analysis output patterns: *.xvg, *.csv, *.png, *.dat
  - If patterns provided, use those instead
  - Return list of found file paths as strings

Results should include: analysis_outputs (dict of script->results), custom_outputs (list), output_files (list of paths).

**scripts/agents/checker.py — CheckerAgent(BaseAgent):**
- `get_role() -> str`: returns "checker"
- `execute(self, input_data: Dict) -> Dict`:
  1. Extract stage and data from input_data
  2. Get validation checks for stage via _get_checks_for_stage()
  3. Run each check, collecting warnings, errors, recommendations
  4. Determine status: FAILURE if errors, NEEDS_REVIEW if warnings, SUCCESS otherwise
  5. If errors found and input_data.get('auto_escalate', False), add recommendation to escalate to debugger
  6. Return HandoffRecord dict

- `_get_checks_for_stage(self, stage: str) -> List[Callable]`:
  - Return list of check functions for the stage from STAGE_CHECKS registry
  - Each check function takes data dict, returns CheckResult(level='error'|'warning'|'info', message=str, recommendation=Optional[str])

- `CheckResult`: Simple dataclass with level, message, recommendation fields

- Built-in check functions (define as module-level or static methods):
  - `check_output_exists(data)`: Verify output_dir exists and has files
  - `check_return_code(data)`: Verify returncode == 0
  - `check_log_errors(data)`: Scan log_file for ERROR/FATAL patterns
  - `check_file_sizes(data)`: Verify output files are non-empty

- `STAGE_CHECKS`: Dict mapping stage names to lists of check functions. Default: all stages get [check_output_exists, check_return_code, check_log_errors, check_file_sizes]. Additional stage-specific checks can be registered.

**scripts/agents/debugger.py — DebuggerAgent(BaseAgent):**
- `get_role() -> str`: returns "debugger"
- `execute(self, input_data: Dict) -> Dict`:
  1. Collect environment info via _capture_environment()
  2. Analyze the error/issue from input_data
  3. Generate debug report with environment, error context, and recommendations
  4. Save debug report to workspace / '.debug_reports' / f'{stage}_{timestamp}.json'
  5. Return HandoffRecord with debug report in data

- `_capture_environment(self) -> Dict[str, Any]`:
  - Capture: Python version, GROMACS version (subprocess: `gmx --version` parse), gnina version (`gnina --version`), gmx_MMPBSA version (`gmx_MMPBSA --version`)
  - Capture: conda env name (from CONDA_DEFAULT_ENV), hostname, working directory, PATH
  - Return dict. For version commands that fail, record "not_available" rather than crashing.

- `_analyze_error(self, input_data: Dict) -> Dict[str, Any]`:
  - Extract error messages from input_data['errors']
  - Check stderr for common GROMACS error patterns (e.g., "Fatal error", "Segmentation fault", "Out of memory")
  - Return dict with: error_type (str), error_messages (list), possible_causes (list of strings), suggested_fixes (list of strings)
  - Use a simple pattern-matching dict for known error types

- `_generate_report(self, environment: Dict, error_analysis: Dict, input_data: Dict) -> Dict`:
  - Combine into structured debug report: timestamp, stage, environment, error_analysis, input_context (sanitized input_data), command_history (from input_data if available)
  - Return report dict

- `KNOWN_ERROR_PATTERNS`: Dict mapping regex patterns to (error_type, possible_causes, suggested_fixes). Include at least:
  - "Fatal error" → GROMACS fatal, check input topology
  - "Segmentation fault" → memory/input issue
  - "command not found" → PATH/environment issue
  - "No such file or directory" → missing input file
