Create OrchestratorAgent extending BaseAgent.

**Class: OrchestratorAgent(BaseAgent)**
- `get_role() -> str`: returns "orchestrator"
- `execute(self, input_data: Dict) -> Dict`: Main orchestration loop for a single stage transition:
  1. Read current stage from input_data['stage'] (as WorkflowStage string value)
  2. Determine target agent via get_agent_for_stage()
  3. Check if previous stage's verification gate passed (if applicable) using VerificationGate
  4. Return handoff record routed to the target agent, with status SUCCESS or BLOCKED (if gate not passed)
  
- `advance_stage(self, current_stage: WorkflowStage) -> Optional[WorkflowStage]`: Returns next stage in sequence (use WorkflowStage enum ordering). Returns None if COMPLETE.

- `create_verification_gate(self, stage: str, description: str, metadata: Dict = None, output_paths: List[str] = None) -> None`: Creates a VerificationGate for the given stage using Phase 1's VerificationGate class. Gate dir = workspace / '.gates'.

- `check_gate(self, stage: str) -> bool`: Checks if stage's verification gate can proceed. Returns True if no gate exists (first run) or gate is approved.

- `get_workflow_status(self) -> Dict[str, Any]`: Returns dict with current_stage (from agent state), completed_stages (list), pending_gates (list of stages with pending gates), last_updated timestamp.

- `initialize_workflow(self) -> Dict[str, Any]`: Sets initial state (stage=INIT, completed_stages=[]), saves checkpoint, returns handoff to start first stage.

Store workflow state using self.state (AgentState) with keys like 'workflow.current_stage', 'workflow.completed_stages'.

Use VerificationGate from scripts.infra.verification for gate management.

<task type="auto">
  <name>Task 2: Implement RunnerAgent</name>
  <files>scripts/agents/runner.py</files>
  <action>
Create RunnerAgent extending BaseAgent.

**Class: RunnerAgent(BaseAgent)**
- `get_role() -> str`: returns "runner"
- `execute(self, input_data: Dict) -> Dict`: 
  1. Extract stage, script path, and params from input_data
  2. Execute script via _run_script()
  3. Extract metrics and warnings from output via _extract_metrics() and _extract_warnings()
  4. Build HandoffRecord with from_agent="runner", to_agent=input_data.get('next_agent', 'orchestrator'), status based on return code
  5. Save checkpoint with stage results
  6. Return handoff dict

- `_run_script(self, script: str, params: Dict[str, Any], env: Dict[str, str] = None) -> Dict[str, Any]`: 
  - Build command list from script path and params (params as --key value CLI flags)
  - Use subprocess.run with cwd=self.workspace, capture_output=True, text=True
  - Optionally merge env vars into os.environ
  - Return dict with: returncode, stdout, stderr, command (str), duration_seconds (time.time delta)

- `_build_command(self, script: str, params: Dict[str, Any]) -> List[str]`: 
  - Converts params dict to CLI flags: --key value pairs
  - Handles boolean flags (just --flag, no value)
  - Returns ['bash', script_path, ...flags] for .sh scripts, ['python', script_path, ...flags] for .py scripts

- `_extract_metrics(self, result: Dict) -> Dict[str, Any]`: 
  - Parse stdout for common patterns: lines containing "RMSD", "energy", "score", numeric summaries
  - Return dict of found metrics (can be empty)
  - This is best-effort extraction, not comprehensive parsing

- `_extract_warnings(self, result: Dict) -> List[str]`: 
  - Scan stderr for WARNING/WARN/Error patterns
  - Return list of warning strings found
  - Also flag non-zero return codes

Do NOT use LocalExecutor/SlurmExecutor here — Runner is for structured script invocation with output shaping, not raw command execution. subprocess.run is appropriate.
