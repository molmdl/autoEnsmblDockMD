  <name>Task 1: Collect per-stage metrics</name>
  <files>.planning/phases/07-first-controlled-execution/07-execution-report.md</files>
  <action>
Aggregate metrics from all handoff records:

1. Parse all handoff records:
   ```bash
   cat > /tmp/collect_metrics.py << 'PYTHON'
   import json
   from pathlib import Path
   from datetime import datetime
   
   workspace = Path("work/test")
   handoffs_dir = workspace / ".handoffs"
   
   metrics = []
   
   for handoff_file in sorted(handoffs_dir.glob("*.json")):
       with open(handoff_file) as f:
           record = json.load(f)
       
       metrics.append({
           'stage': record.get('stage', 'unknown'),
           'status': record.get('status', 'unknown'),
           'duration_seconds': record.get('duration_seconds', 0),
           'timestamp': record.get('timestamp', ''),
           'warnings_count': len(record.get('warnings', [])),
           'errors_count': len(record.get('errors', [])),
           'file': handoff_file.name
       })
   
   # Print as JSON
   print(json.dumps(metrics, indent=2))
   PYTHON
   
   python /tmp/collect_metrics.py > /tmp/stage_metrics.json
   ```

2. Calculate cumulative metrics:
   ```bash
   cat > /tmp/calculate_analytics.py << 'PYTHON'
   import json
   
   with open('/tmp/stage_metrics.json') as f:
       metrics = json.load(f)
   
   total_duration = sum(m['duration_seconds'] for m in metrics)
   success_count = sum(1 for m in metrics if m['status'] == 'success')
   failure_count = sum(1 for m in metrics if m['status'] == 'failure')
   needs_review_count = sum(1 for m in metrics if m['status'] == 'needs_review')
   total_warnings = sum(m['warnings_count'] for m in metrics)
   total_errors = sum(m['errors_count'] for m in metrics)
   
   analytics = {
       'total_duration_seconds': total_duration,
       'total_duration_hours': total_duration / 3600,
       'stage_count': len(metrics),
       'success_count': success_count,
       'failure_count': failure_count,
       'needs_review_count': needs_review_count,
       'total_warnings': total_warnings,
       'total_errors': total_errors,
       'stages': metrics
   }
   
   print(json.dumps(analytics, indent=2))
   PYTHON
   
   python /tmp/calculate_analytics.py > /tmp/analytics.json
   ```

3. Calculate disk usage:
   ```bash
   cat > /tmp/disk_usage.txt << 'EOF'
   ## Disk Usage
   
   EOF
   
   du -sh work/test >> /tmp/disk_usage.txt
   echo "" >> /tmp/disk_usage.txt
   echo "### Breakdown by Stage" >> /tmp/disk_usage.txt
   du -sh work/test/rec work/test/dock work/test/com 2>/dev/null >> /tmp/disk_usage.txt
   echo "" >> /tmp/disk_usage.txt
   echo "### Growth During Execution" >> /tmp/disk_usage.txt
   echo "Initial: (not tracked - would need checkpoint at start)" >> /tmp/disk_usage.txt
   echo "Final: $(du -sh work/test | cut -f1)" >> /tmp/disk_usage.txt
   ```

4. Document token tracking (if applicable):
   ```bash
   # Note: Token tracking is external to pipeline scripts
   # If this was an agent-assisted execution, tokens would be tracked via OpenCode
   cat > /tmp/token_tracking.txt << 'EOF'
   ## Token Tracking
   
   **Mode:** Manual execution (not agent-assisted)
   
   Token tracking is not applicable for manual pipeline execution.
   If agent-assisted execution was used, tokens would be tracked via:
   - OpenCode session metadata
   - LLM API usage logs
   - Cache hit/miss statistics
   
   For future agent-assisted runs, consider implementing:
   - Wrapper-level token counting
   - Session-based token budget tracking
   - Per-stage token attribution
   EOF
   ```

DO NOT fabricate token counts if manual execution was used.
DO calculate metrics from actual handoff records.
