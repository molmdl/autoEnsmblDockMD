Create `.opencode/docs/AGENT-WORKFLOW.md` structured for agent parsing (not human narrative). Include these sections:

1. **Pipeline State Machine** — Define all stages as an ordered state machine:
   - `receptor_prep` → `receptor_md` → `receptor_cluster` → `receptor_align` → `docking` → `complex_setup` → `complex_md` → `mmpbsa` → `analysis`
   - For each state: name, slash command, skill file path, script path
   - Valid transitions (which states can follow which)

2. **HandoffRecord Schema** — Full JSON schema for the handoff record:
   - Fields: `stage`, `status` (pending|running|complete|failed), `inputs` (dict), `outputs` (dict), `next_action`, `error` (optional), `timestamp`
   - Example JSON for a completed docking stage handing off to complex_setup

3. **Decision Trees** — Agent logic for:
   - "Given current stage + status, what is the next action?" (table format)
   - "Stage failed → invoke debugger-diagnose → retry or escalate?"
   - "Stage complete → invoke checker-validate → proceed or block?"
   - "All stages complete → generate final report"

4. **Stage Preconditions/Postconditions** — For each stage:
   - Required input files (glob patterns)
   - Required environment (GROMACS, gnina, etc.)
   - Expected output files (glob patterns)
   - Validation checks (what checker-validate looks for)

5. **Orchestration Protocol** — Step-by-step for an orchestrator agent:
   - Load last HandoffRecord (or initialize new run)
   - Determine current stage from state
   - Load appropriate skill via `@.opencode/skills/{skill}/SKILL.md`
   - Invoke slash command with parameters from handoff
   - Capture outputs, update HandoffRecord
   - Run checker-validate
   - If pass → advance state; if fail → invoke debugger-diagnose
   - Write updated HandoffRecord

Use YAML/JSON code blocks for schemas. Use tables for state transitions and decision logic. Avoid narrative prose — use structured lists and tables throughout. Cross-reference SKILL.md files by exact path. Cross-reference scripts/commands/*.sh by exact path.

Target: ~200-300 lines. Dense, parseable, no fluff.
