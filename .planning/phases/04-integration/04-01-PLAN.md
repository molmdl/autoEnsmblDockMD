  <action>
Create `scripts/commands/common.sh` with these functions:

1. `find_workspace_root()` — Walk up from CWD looking for `config.ini` or `.gsd-workspace` marker. Fall back to CWD if not found. Print path to stdout.

2. `ensure_env()` — Source `${PROJECT_ROOT}/scripts/setenv.sh` if it exists. PROJECT_ROOT is computed as `$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)`.

3. `parse_flags()` — Accept positional args. Set global vars: `CONFIG` (from --config, default "config.ini"), `VERBOSE` (from --verbose, default false), `EXTRA_PARAMS` as JSON object (from any --key value pairs, using jq). Print usage on --help.

4. `dispatch_agent()` — Accept agent name (e.g. "runner") and stage name. Build JSON handoff from CONFIG + EXTRA_PARAMS. Call `python -m scripts.agents --agent $AGENT --input <(echo "$HANDOFF_INPUT")`. Return exit code.

5. `check_handoff_result()` — Accept stage name. Read `.handoffs/${stage}.json`. Print status. If failure: print errors + recommendations to stderr, exit 1. If needs_review: print warnings, exit 2. If success: exit 0.

Use `set -euo pipefail`. Add header comment explaining this is sourced by command scripts.

Do NOT use `conda activate` directly — that's handled by setenv.sh.

#!/usr/bin/env bash
# scripts/commands/{name}.sh - Bridge for /{name} slash command
set -euo pipefail

source "${SCRIPT_DIR}/common.sh"

ensure_env
WORKSPACE_ROOT="$(find_workspace_root)"
cd "${WORKSPACE_ROOT}"
parse_flags "$@"

dispatch_agent "{agent}" "{stage}"
check_handoff_result "{stage}"
