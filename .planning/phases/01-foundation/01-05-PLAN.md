```python
from enum import Enum

class GateState(Enum):
    """Verification gate states."""
    PENDING = 'pending'
    APPROVED = 'approved'
    REJECTED = 'rejected'
    PAUSED = 'paused'

class VerificationGate:
    """Human verification gate mechanism."""
    
    def __init__(self, stage_name: str, gate_dir: str = '.gates'):
        """Initialize gate for specific stage."""
        
    def create_gate(self, description: str, metadata: Dict[str, Any] = None,
                    output_paths: List[str] = None, metrics: Dict[str, Any] = None) -> None:
        """Create a new verification gate with context for human review."""
        
    def approve(self, approver: str = 'human', notes: str = '') -> bool:
        """Approve the gate and record decision."""
        
    def reject(self, reason: str, rejecter: str = 'human') -> bool:
        """Reject the gate with reason."""
        
    def pause(self, reason: str = '') -> bool:
        """Pause at current gate for later review."""
        
    def get_state(self) -> GateState:
        """Get current gate state."""
        
    def can_proceed(self) -> bool:
        """Check if workflow can proceed past this gate."""
        
    def get_gate_info(self) -> Optional[Dict[str, Any]]:
        """Get full gate information including history."""
        
    def add_note(self, note: str) -> bool:
        """Add a note to the gate without changing state."""
        
    def get_context_summary(self) -> str:
        """Get formatted summary for human review."""
```

Implementation requirements:

1. **Gate file format (JSON):**
   ```json
   {
       "stage": "docking",
       "state": "pending",
       "created_at": "2026-03-24T14:30:22",
       "description": "Review docking results before MD",
       "metadata": {"ligand_count": 5, "best_score": -8.5},
       "output_paths": ["/path/to/docked_poses.pdb"],
       "metrics": {"rmsd": 2.3, "success_rate": 0.8},
       "history": [
           {"timestamp": "...", "action": "created", "actor": "system"}
       ]
   }
   ```

2. **State transitions:**
   - PENDING → APPROVED (approve)
   - PENDING → REJECTED (reject)
   - PENDING → PAUSED (pause)
   - PAUSED → APPROVED (approve)
   - PAUSED → REJECTED (reject)
   - Invalid transitions should raise ValueError

3. **History tracking:**
   - Every action appends to history list
   - Record: timestamp, action, actor, notes/reason

4. **get_context_summary() format:**
   ```
   === Verification Gate: docking ===
   State: PENDING
   Created: 2026-03-24 14:30:22
   
   Description: Review docking results before MD
   
   Output Files:
   - /path/to/docked_poses.pdb
   
   Key Metrics:
   - rmsd: 2.3
   - success_rate: 0.8
   
   History:
   - 2026-03-24 14:30:22: Gate created by system
   ```

5. **CLI interface:**
   - `python -m scripts.infra.verification create --stage NAME --desc "Description"`
   - `python -m scripts.infra.verification status --stage NAME`
   - `python -m scripts.infra.verification approve --stage NAME --notes "Looks good"`
   - `python -m scripts.infra.verification reject --stage NAME --reason "Need rerun"`
   - `python -m scripts.infra.verification pause --stage NAME --reason "Waiting for input"`
   - `python -m scripts.infra.verification summary --stage NAME`
   - `python -m scripts.infra.verification list` (list all gates)

6. **File locking for concurrent access:**
   - Use fcntl.flock for POSIX systems
   - Acquire exclusive lock on write, shared lock on read
   - Timeout after 5 seconds

7. **Gate directory management:**
   - Create gate_dir if it doesn't exist
   - Gate filename: `{stage_name}_gate.json`

DO NOT:
- Allow bypassing gates (enforce state checks)
- Auto-approve gates (explicit human action required)
- Delete gate files on approval (keep for audit trail)
- Require all metadata fields (description is mandatory, others optional)
