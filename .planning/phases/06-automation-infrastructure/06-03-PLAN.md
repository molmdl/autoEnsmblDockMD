  <action>
Create `scripts/infra/plugins/handoff_inspect.py` following "Handoff Inspector with Status Normalization" pattern from 06-RESEARCH.md.

Function `inspect_latest_handoff(workspace: Path) -> HandoffRecord`:
1. Find .handoffs/ directory in workspace
2. If no handoffs exist: return SUCCESS with warning "No handoffs found (new workspace)", next_action = "Run workspace initialization"
3. If handoffs exist:
   - Glob for *.json files
   - Find latest by st_mtime: `latest = max(handoffs, key=lambda p: p.stat().st_mtime)`
   - Parse JSON and extract: stage, status, errors, warnings, recommendations
   - Normalize status vocabulary using map:
     ```python
     status_map = {
         "success": "SUCCESS",
         "failure": "FAILED",
         "needs_review": "NEEDS_REVIEW",
         "blocked": "BLOCKED"
     }
     ```
   - Provide next-action guidance:
     - If SUCCESS: "Stage {stage} completed successfully. Proceed to next stage."
     - If NEEDS_REVIEW: "Stage {stage} needs review. Check warnings: {warnings}"
     - If FAILED/BLOCKED: "Stage {stage} failed. Review errors: {errors}"

4. Return HandoffRecord with:
   - from_agent: "handoff_inspector"
   - to_agent: "orchestrator"
   - stage: "handoff_inspection"
   - status: SUCCESS (inspector always succeeds unless workspace unreadable)
   - data: {latest_stage, latest_status, handoff_file, next_action}
   - warnings/errors: propagated from inspected handoff if status is NEEDS_REVIEW/FAILED

main() function:
- argparse: --workspace (required)
- Call inspect_latest_handoff(Path(workspace))
- Print record.to_dict() as JSON
- Exit 0 (inspector tool itself should always succeed even if latest handoff failed)

Docstring: "Handoff inspector plugin - parses latest handoff and provides next-action guidance."
