  <action>
Create `scripts/infra/plugins/preflight.py` following "Preflight Validation with Tiered Severity" pattern from 06-RESEARCH.md.

PreflightValidator class with methods:
1. `__init__(self, config_path: Path)` - store path, create HandoffRecord
2. `validate(self) -> HandoffRecord` - run all checks, determine final status
3. `check_config_exists(self)` - ERROR if config.ini missing
4. `check_required_sections(self)` - ERROR if missing [general], [docking], [receptor], [complex]
5. `check_mode_specific_requirements(self)` - Mode-aware validation:
   - Parse [docking] mode value (targeted/blind/test)
   - If mode=targeted: ERROR if reference_ligand not specified, WARNING if reference file missing
   - If mode=blind: INFO that autobox will be used
6. `check_tool_availability(self)` - WARNING if gmx/gnina/gmx_MMPBSA not in PATH (use shutil.which)
7. `check_input_files(self)` - WARNING if work/input/ directory empty or missing receptor.pdb

Use configparser.ConfigParser to parse INI file (Python stdlib, no external deps).

Severity tier logic:
- If ANY errors: HandoffStatus.FAILURE
- If NO errors but warnings: HandoffStatus.NEEDS_REVIEW
- If NO errors/warnings: HandoffStatus.SUCCESS

main() function:
- argparse: --config (required), --workspace (optional, default to cwd)
- Call PreflightValidator(config_path).validate()
- Print record.to_dict() as JSON to stdout
- Exit 0 if SUCCESS/NEEDS_REVIEW, exit 1 if FAILURE/BLOCKED

Include docstring: "Preflight validation plugin - validates config, tools, and inputs before workflow execution."
