## Critical Best Practices Checklist

For EACH skill, check:

### YAML Frontmatter (per agentskills.io spec)
- [ ] Has `name` field (letters, numbers, hyphens only - no parentheses)
- [ ] Has `description` field (max 1024 chars total frontmatter)
- [ ] Description starts with "Use when..." (CSO requirement)
- [ ] Description describes TRIGGERS, not workflow (critical CSO principle)
- [ ] Description written in third person

### Claude Search Optimization (CSO)
- [ ] Keywords throughout for search (error messages, symptoms, tools)
- [ ] Name uses active voice, verb-first (e.g., "creating-skills" not "skill-creation")
- [ ] Description includes concrete triggers and symptoms
- [ ] No workflow summary in description (agents must read full skill)

### Structure
- [ ] Overview section with core principle (1-2 sentences)
- [ ] "When to Use" section with triggers
- [ ] Quick reference (table or bullets)
- [ ] Implementation section
- [ ] Common mistakes section

### Token Efficiency
- [ ] Skill under 500 words (unless heavy reference type)
- [ ] One excellent example (not multi-language)
- [ ] No narrative storytelling
- [ ] Cross-references use skill name only (not @ syntax - burns context)
